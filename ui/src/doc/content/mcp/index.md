# MCP 服务管理

MCP（Model Context Protocol）管理是智能体的"外部能力接入中枢"，通过标准化协议将异构的外部工具、服务、数据源统一封装为智能体可理解和调用的能力，实现"思考-行动"闭环。


## 功能概述

- 支持三种协议类型：HTTP、SSE、STDIO
- 支持同步和异步两种运行模式
- 手动连接校验连通性并加载工具目录，支持刷新工具目录
- 工具治理：按工具维度控制"全局可用"状态，精细管理工具暴露范围
- 工具调试：全屏调试面板，基于 JSON Schema 动态表单，支持历史记录持久化
- 运行时自动降级：连续失败自动标记为连接失败，避免不可用服务持续参与调用
- 智能体关联管理：一个智能体可关联多个 MCP 服务器，删除/停用时自动检测引用
- 卡片状态角标：直观展示停用、待连接、连接中、已激活、连接失败等状态


## 核心概念

### 协议类型

| 协议 | 说明 | 配置项 |
|------|------|--------|
| **HTTP** | 标准 HTTP 请求调用 MCP 服务，适用于简单的请求-响应模式 | URL、请求头、查询参数 |
| **SSE** | 基于 Server-Sent Events 的流式连接，适用于需要服务端推送或长连接的场景 | URL、请求头、查询参数 |
| **STDIO** | 通过标准输入输出与本地进程通信 | 命令、命令参数、环境变量、工作目录、字符编码 |

### 运行模式

| 模式 | 说明 |
|------|------|
| **同步（SYNC）** | 工具调用同步等待结果返回 |
| **异步（ASYNC）** | 工具调用异步返回结果 |

### 连接状态

每个 MCP 服务器有四种连接状态，决定其是否能参与智能体调用：

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| **待连接** | `NOT_ACTIVATED` | 已创建但尚未执行连接操作，工具目录未加载 |
| **连接中** | `ACTIVATING` | 正在建立连接并加载工具目录，显示加载动画 |
| **已激活** | `ACTIVE` | 连接成功且工具目录已就绪，可参与智能体调用 |
| **连接失败** | `FAILED` | 连接异常或超时，需排查后通过「重试连接」恢复 |

### 卡片角标状态

MCP 服务卡片头像右下角的角标直观展示服务可用性：

| 角标 | 触发条件 | 说明 |
|------|----------|------|
| 无角标 | 停用 / 连接中 | 灰色头像或加载动画，无需额外标识 |
| 黄色叹号 | 待连接 / 已连接但无工具 | 提示需执行连接操作或工具目录为空 |
| 红色叉号 | 连接失败 | 连接异常，需重试 |
| 绿色对号 | 已激活且有工具 | 服务可用 |

悬停角标可查看详细提示，已激活状态下显示工具总数和全局可用数。

### 工具治理

MCP 连接成功后会加载工具目录，每个工具具有独立的"全局可用"开关：

- **全局可用（enabled）**：运行时只会注册全局可用且未消失的工具
- **已消失（missing）**：工具在 MCP 服务器端已被移除，本地仍保留记录但无法使用
- 运行时自动降级期间，工具治理面板进入只读态，仅可查看不可修改

### 运行时自动降级

当 MCP 服务在运行时（非手动操作）连续出现传输类异常（连接超时、连接重置、管道断开等），系统会自动将其标记为连接失败，避免不可用服务持续参与智能体调用。

- **自动降级失败次数**：可在创建/编辑时配置，默认为 3 次；设为 0 表示关闭自动降级
- **失败分类**：仅传输层异常（如 `SocketTimeoutException`、`ConnectException`、`ConnectionReset` 等）计入降级，业务异常不触发
- **恢复方式**：通过菜单中的「重试连接」操作手动恢复


## 操作指南

### 创建 MCP 服务器

1. 在 MCP 管理页面点击 **添加新 MCP 服务器** 卡片
2. 根据当前协议筛选状态，创建方式有所不同：
   - **全部协议**：在卡片内选择要创建的协议类型（HTTP / SSE / STDIO）
   - **已筛选协议**：直接点击卡片创建对应协议的服务器

3. 填写表单：
   - **名称**：服务器名称（必填）
   - **描述**：服务器功能说明（必填，不超过 200 字符）
   - **协议类型**：HTTP / SSE / STDIO（创建后不可修改）
   - **运行模式**：同步 / 异步
   - **超时时间**：调用超时时间，单位毫秒，默认 30000ms，最小 1000ms
   - **自动降级失败次数**：连续失败后自动标记为连接失败的阈值，默认 3 次；设为 0 关闭

4. HTTP / SSE 协议配置：
   - **URL**：服务地址
   - **查询参数**：URL 查询参数键值对，支持多条
   - **请求头**：自定义请求头键值对，支持多条

5. STDIO 协议配置：
   - **命令**：可执行命令路径
   - **命令参数**：命令行参数列表，支持多条
   - **环境变量**：运行时环境变量键值对，支持多条
   - **工作目录**（可选）：进程工作目录路径
   - **字符编码**（可选）：默认 UTF-8

创建成功后，需手动执行「连接」操作来校验连通性并加载工具目录。


### 连接与刷新工具

连接和刷新工具是 MCP 管理的两个核心操作：

- **连接**：建立与 MCP 服务器的通信连接，校验连通性并加载完整的工具目录。待连接或连接失败的服务器需执行此操作。
- **刷新工具**：在已有连接的基础上，重新获取工具目录。当 MCP 服务器端工具发生变更时使用。

:::tip
启用决定 MCP 是否允许参与系统；连接用于校验连通性并加载工具目录；刷新工具只在当前配置基础上重新获取工具目录。
:::

操作方式：点击卡片右上角菜单 → **连接** / **重试连接** / **刷新工具**。

### 启用与停用

- 点击卡片右上角菜单 → **启用/停用** 切换 MCP 服务器的启用状态
- 停用的 MCP 服务器不会参与智能体调用
- 若该服务器正在被智能体引用，停用时将弹出二次确认提示，显示引用的智能体名称
- 启用决定 MCP 是否允许参与系统，与连接状态无关

:::tip
启用/停用和连接/断开是两个独立操作：启用/停用控制是否允许参与智能体调用，连接/断开控制是否已建立通信连接并加载工具目录。
:::

### 查看详情

点击卡片右上角菜单 → **查看**，以弹窗方式查看完整配置信息，包括：

- 关联智能体、名称、协议、运行模式
- 启用状态、超时时间、自动降级失败次数
- 连接状态、状态说明、失败来源
- 工具总数、全局可用工具数、待刷新标记
- 连接状态变更时间、上次连接时间、上次工具刷新时间
- 健康状态、上次健康检查时间
- 协议配置详情（JSON 格式）

### 编辑配置

点击卡片右上角菜单 → **编辑**，修改服务器配置。编辑模式下可查看关联的智能体列表。

:::warning
修改 MCP 服务器配置后，如果该服务此前处于已激活状态，系统会自动尝试重连。重连失败时会显示警告提示，但不影响配置保存。
:::

### 删除服务器

点击卡片右上角菜单 → **删除**：

- 若该服务器正在被智能体引用，会弹出引用提示确认框，显示引用的智能体名称
- 若无引用，弹出常规删除确认框
- 删除操作不可恢复

:::warning
删除被智能体引用的 MCP 服务器时，系统会显示「该 MCP 服务正在被 [xxx] 智能体引用，删除后可能会影响上述智能体的正常使用」。
:::

### 搜索与筛选

- **协议筛选**：页面顶部分段控制器切换协议类型（全部 / HTTP / SSE / STDIO），切换后自动重新加载列表
- **名称搜索**：输入关键词后回车或点击搜索图标搜索，支持按 MCP 服务名称模糊匹配


## 工具治理

在卡片右上角菜单中点击 **工具治理**，打开全屏工具治理面板。

### 功能说明

- 展示该 MCP 服务器下的完整工具目录
- 每个工具以卡片形式展示，包含名称、描述、状态标签（全局可用 / 全局禁用 / 已消失）
- 通过开关控制每个工具的"全局可用"状态
- 支持按工具名称或描述关键词搜索

### 工具状态

| 状态 | 说明 |
|------|------|
| **全局可用** | 运行时正常注册该工具，智能体可调用 |
| **全局禁用** | 工具存在但不注册，智能体不可调用 |
| **已消失** | 工具在 MCP 服务器端已被移除，无法使用 |

### 只读态

当 MCP 服务器因运行时自动降级而处于连接失败状态时，工具治理面板进入只读态：
- 顶部显示警告提示："当前展示为上次缓存"
- 所有工具的启停开关不可操作
- 调试按钮不可用
- 需通过「重试连接」恢复正常后才可修改


## 工具调试

从工具治理面板点击工具卡片上的调试按钮（虫子图标），进入全屏 MCP 调试面板。

### 调试面板布局

调试面板采用三栏布局：

- **顶部工具栏**：退出按钮、服务器名称、工具切换下拉框
- **左侧边栏**：调试历史记录列表，支持查看、删除单条、清空全部
- **右侧主区域**：调试表单 + 执行结果，左右分栏展示

### 调试流程

1. **选择工具**：通过顶部下拉框切换当前调试工具（仅显示可用且未消失的工具）
2. **填写参数**：基于工具的 JSON Schema 自动渲染动态参数表单
   - 支持表单模式和 JSON 编辑模式切换
   - 表单模式自动识别字段类型：字符串、数字、布尔值、枚举、对象/数组
   - 必填字段标红色星号标记
   - 若工具无需参数，显示"此工具无需输入参数"
3. **执行调试**：点击「执行调试」按钮，等待结果返回
4. **查看结果**：右侧面板显示调用结果（成功/失败、耗时、返回内容/错误信息）

### 调试历史

- 每次调试调用（无论成功或失败）自动记录到历史
- 历史记录持久化存储在浏览器 localStorage 中
- 最多保留 20 条记录
- 单条结果内容超过 8000 字符自动截断
- 整体存储体积上限约 1.5MB，超限时自动淘汰最旧记录
- 点击历史记录可查看完整的调用参数和返回结果
- 支持 ESC 键快速退出调试面板


## 在智能体中使用

在智能体配置的「知识库与 MCP」步骤中：

1. 在 **MCP 服务器** 区域，按协议类型折叠面板浏览
2. 勾选需要关联的 MCP 服务器（Checkbox 多选）
3. 关联后，智能体运行时将自动注册已激活且全局可用的工具

:::tip
删除或停用被智能体引用的 MCP 服务器时，系统会显示引用该服务的智能体名称，帮助评估影响范围。
:::

## 常见问题

### HTTP 和 SSE 协议如何选择？

- **HTTP**：适用于简单的请求-响应模式，服务端无需保持连接
- **SSE**：适用于需要服务端推送或长连接的场景

### STDIO 模式的安全注意事项

- STDIO 模式会在服务器本地启动进程
- 确保命令路径安全，避免执行危险命令
- 环境变量中不要包含敏感信息（使用平台的环境变量管理）

### 超时时间如何设置？

- 建议根据工具的响应时间合理设置
- 简单查询类工具：5000 ~ 10000ms
- 复杂处理类工具：30000 ~ 60000ms
- 长时间任务：可使用异步模式

### "连接"和"刷新工具"有什么区别？

- **连接**：重新建立通信连接并加载完整的工具目录，适用于首次接入或连接失败后恢复
- **刷新工具**：在已有连接基础上重新获取工具目录，适用于 MCP 服务器端工具发生变更后同步

### 自动降级失败次数应该设多少？

- 默认 3 次适合大多数场景
- 网络不稳定的环境可适当增大（如 5 次）
- 设置为 0 可关闭自动降级，但可能导致不可用的服务持续参与调用

### 工具治理中显示“已消失”怎么处理？

- “已消失”表示该工具在 MCP 服务器端已被移除，但本地仍有历史记录
- 刷新工具目录后，已消失的工具仍会保留记录但标记为不可用
- 如确认不再需要，可通过重新连接清理过期记录

### 连接失败后如何排查？

- 进入 MCP 详情查看「状态说明」和「失败来源」字段
- 若失败来源为「运行时自动降级」，说明是运行时连续出错导致自动降级
- 检查 URL 是否正确、服务是否可达、认证信息是否有效
- 对于 STDIO 协议，确认本地运行环境是否正确（命令路径、环境变量、工作目录）
- 排查完成后点击「重试连接」恢复

### 删除 MCP 服务器会影响已关联的智能体吗？

- 删除时系统会弹出二次确认框，显示引用的智能体名称
- 确认删除后，关联智能体将无法调用该 MCP 服务器的工具
- 建议先在智能体配置中解除 MCP 关联，再执行删除
