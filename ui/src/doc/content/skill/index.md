# 技能包管理

技能包是将相关工具、参考文档、示例和脚本打包为可复用技能单元的管理机制，赋予智能体完成特定领域复杂任务的能力。技能包以**文件目录**形式组织，支持在平台内置编辑器中直接管理文件结构。


## 功能概述

- 基于文件目录的技能包结构，支持在线编辑与管理
- 左侧文件树 + 右侧编辑器的双栏交互模式
- 支持新建文件/文件夹、上传文件、右键菜单操作
- 文件类型白名单管控，非文本文件引导下载查看
- 支持三种导入模式：本地装载、ZIP 压缩包、Git 仓库
- 技能包可关联多个工具，运行时自动注册到工具集
- SKILL.md 元信息与数据库自动双向同步
- 技能包启用/禁用管理，被引用时二次确认保护
- 分类筛选、关键词搜索与无限滚动分页
- 技能包整体下载为 ZIP 压缩包
- 支持手动同步技能文件到运行节点（集群环境）


## 技能包列表页

技能包管理页面提供以下核心交互能力：

### 分类筛选

页面顶部的分段选择器（Segmented）展示所有已有分类及「全部」选项，切换分类后列表自动刷新。分类数据来源为数据库中所有技能包的 `category` 字段去重结果。

### 关键词搜索

在搜索框中输入技能包名称后按回车或点击搜索按钮，系统按名称模糊匹配并刷新列表。

### 无限滚动分页

列表采用无限滚动加载方式，每页加载 50 条记录。滚动到底部时自动加载下一页，直到所有数据加载完毕。

### 技能包卡片

每个技能包以卡片形式展示，包含以下信息：

| 元素 | 说明 |
|------|------|
| 头像 | 技能包图标，禁用态显示为灰色 |
| 工具角标 | 若已关联工具，头像右下角显示工具图标角标，悬停提示关联数量 |
| 名称 | 点击可查看详情 |
| 描述 | 最多显示三行，超出截断 |
| 分类标签 | 显示当前分类，未设置时显示「未设置标签」 |
| 更新时间 | 卡片底部右侧展示 |
| 操作菜单 | 右上角 `...` 按钮，悬停展开菜单 |

### 操作菜单

卡片右上角菜单提供以下操作（需要 `TENANT_EDITOR` / `TENANT_ADMIN` / `TENANT_OWNER` 角色）：

| 操作 | 说明 |
|------|------|
| 查看 | 弹窗展示技能包详情（关联智能体、分类、名称、描述、是否关联工具） |
| 编辑 | 进入技能包编辑器 |
| 分类 | 打开分类选择器，支持搜索已有分类或新建分类 |
| 关联工具 | 打开关联工具选择器弹窗 |
| 启用/禁用 | 切换技能包状态；若被智能体引用且当前为启用，弹出二次确认 |
| 删除 | 删除技能包；若被智能体引用，弹出二次确认提示 |

### 启用/禁用

启用/禁用操作直接影响智能体运行时是否加载该技能包。当技能包正在被智能体引用且当前处于启用状态时，系统会弹出确认框，提示「该技能包正在被 [xxx] 智能体引用，禁用后可能会影响上述智能体的正常使用」。

### 删除保护

删除操作会同时清理以下数据：
- 数据库中的技能包记录
- 技能包与智能体的关联关系
- 技能包与工具的关联关系
- 文件系统中的技能包目录
- 数据库中的技能文件记录

若技能包正在被智能体引用，系统会弹出二次确认框。


## 技能包目录结构

每个技能包在文件系统中为一个独立目录，标准结构如下：

```
<技能包名称>/
  SKILL.md            # 技能定义文件（必需），含 YAML frontmatter
  scripts/            # 可执行脚本目录
  examples/           # 示例文件目录
  references/         # 参考文档目录
```

### SKILL.md 文件

`SKILL.md` 是技能包的元信息文件，须包含 YAML frontmatter：

```yaml
---
name: 技能包名称
description: 技能包描述
---
# 正文内容（Markdown）
```

- `name` 字段作为技能包唯一标识，修改后自动同步到数据库
- `description` 字段描述技能功能，修改后自动同步到数据库
- 正文部分使用 Markdown 编写，用于描述技能能力、使用场景和操作步骤
- SKILL.md 不允许删除，不支持右键菜单操作

### 附属目录

| 目录 | 用途 | 说明 |
|------|------|------|
| `scripts/` | 可执行脚本 | 存放 Python、Shell、JavaScript 等脚本文件 |
| `examples/` | 示例文件 | 存放技能使用示例和演示文件 |
| `references/` | 参考文档 | 存放参考文档、知识片段等 |

以上三个目录及其下的文本文件会自动入库管理；也支持在根目录或任意子目录下创建自定义文件。


## 关联工具

### 关联工具选择器

通过卡片菜单的「关联工具」打开选择器弹窗，采用**左右双面板**布局：

- **左侧面板（可选工具）**：展示系统中所有已启用的工具，显示名称、分类标签和描述
- **右侧面板（已选工具）**：展示当前技能包已关联的工具

两侧均支持关键词搜索（按名称、描述、分类过滤），点击工具即可在左右面板间切换添加/移除。

### 运行时自动注册

技能包关联的工具会在智能体运行时自动注册到工具集。具体流程：

1. 智能体启动时，根据关联的技能包 ID 列表加载所有**已启用**的技能包
2. 对每个技能包，查询其关联的工具 ID 列表
3. 将工具配置注册到智能体的 Toolkit 中
4. 同时在 SKILL.md 内容末尾追加可用工具信息，帮助模型感知可用工具

因此，只需在技能包中关联工具，智能体选中该技能包后即可自动获得工具调用能力。


## 创建与管理技能包

### 手动创建

1. 在技能包管理页面点击 **添加新技能包** 卡片（需 `TENANT_EDITOR` / `TENANT_ADMIN` / `TENANT_OWNER` 角色）
2. 填写基本信息：名称、描述、分类
3. 点击创建后自动进入技能包编辑器，系统已初始化 `SKILL.md` 文件

### 技能包编辑器

编辑器采用**左侧文件树 + 右侧编辑器**的双栏布局，左侧面板宽度可通过拖拽分割条自由调整（范围 240px ~ 500px）。

**工具栏**（左侧面板顶部）：
- **返回** — 返回技能包列表（有未保存修改时弹出确认提示）
- **新建文件**（+）— 在根目录或选定目录下创建文件，创建时校验文件扩展名是否在白名单内
- **新建文件夹**（文件夹图标）— 创建目录，提供 `scripts`、`references`、`examples` 快捷预设
- **下载技能包** — 将整个技能包目录打包为 ZIP 下载

**底部操作栏**（左侧面板底部）：
- **同步到运行节点** — 手动触发技能文件同步到 runner-file 节点（集群环境，详见「文件同步机制」章节）
- **删除技能包** — 删除技能包并返回列表

**文件树操作**（右键菜单）：
- **新建文件** / **新建文件夹** — 在指定目录下创建（空白区域右键可在根目录创建）
- **上传文件** — 上传本地文件到当前目录，文本文件自动入库，二进制文件直接落盘
- **删除** — 删除文件或文件夹（目录删除时自动清理数据库关联记录，递归删除子文件）
- **注意**：`SKILL.md` 不允许删除，不显示右键菜单

**文件管理特性**：
- 文件树默认全部展开，方便浏览完整结构
- 删除当前正在编辑的文件后，编辑区自动清空
- 文件类型在白名单内的可直接在线编辑（如 `.md`、`.py`、`.js`、`.json` 等）
- 非白名单文件类型（如图片、PDF、二进制文件）点击时弹出确认框引导下载到本地查看
- 超大文件（>500KB）点击时弹出确认框引导下载到本地查看
- 未保存修改时切换文件会弹出确认提示（可选择放弃更改或继续编辑）
- 未保存修改时关闭/刷新浏览器页面会弹出浏览器原生确认提示
- 文件节点右侧显示保存状态图标（绿色对勾=已同步，黄色编辑图标=有未同步更改）

### 编辑 SKILL.md

1. 在文件树中点击 `SKILL.md`（新建技能包后自动选中）
2. 在右侧 CodeMirror Markdown 编辑器中进行编辑
3. 保存后，系统自动解析 YAML frontmatter 中的 `name` 和 `description`，同步更新数据库中的技能包元信息

编辑器顶部显示当前文件路径面包屑，工具栏提供以下操作：
- **保存** — 保存文件内容（支持 Ctrl+S 快捷键），入库文件同时更新数据库和文件系统，非入库文件仅写入文件系统
- **放弃更改** — 恢复到上次保存的内容
- **已保存指示器** — 绿色对勾表示当前内容已保存

### 设置分类

在技能包卡片右上角菜单中点击 **分类**，弹出分类选择器：

- 从已有分类中选择
- 支持搜索过滤
- 支持新建分类：在分隔线下方输入新分类名称后点击添加
- 设置成功后自动刷新列表

## 导入技能包

点击技能包列表页的「添加新技能包」卡片，可看到三种导入入口：

### 导入方式对比

| 方式 | 入口文案 | 说明 | 适用场景 |
|------|----------|------|----------|
| **本地装载** | 装载本地技能包 | 指定服务器本地目录路径，直接扫描并导入 | 已有技能包目录、开发调试 |
| **ZIP 导入** | 导入技能压缩包 | 上传 ZIP 压缩包（≤100MB），自动解压并导入 | 离线分享、批量导入 |
| **Git 导入** | 下载 Git 技能包 | 从 Git 仓库克隆并导入（支持 HTTPS / SSH） | 版本管理、团队协作 |

### 覆盖策略

三种导入方式均支持覆盖策略开关：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| **跳过**（默认） | 同名技能包保留已有内容，跳过导入 | 防止误覆盖、保留本地修改 |
| **覆盖** | 删除旧文件记录和目录后导入新内容 | 版本更新、全量同步 |

### 导入结果提示

导入完成后，系统会根据结果展示不同的提示信息：

| 场景 | 提示内容 |
|------|----------|
| 未识别到技能包 | 弹窗展示诊断信息（如目录结构不正确） |
| 全部跳过 | 提示「均已存在且未开启覆盖」 |
| 部分跳过 | 提示「成功导入 N 个，跳过 M 个同名技能」 |
| 全部成功 | 提示「成功导入 N 个技能包」 |

导入成功后，列表自动切换到导入时指定的分类并刷新数据。


## 本地装载详解

本地装载从服务器本地目录直接导入技能包，默认路径为 `.apboa/skills`。

**表单字段：**

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 技能分类 | 导入的技能包归入此分类 | `本地导入` |
| 本地路径 | 服务端可访问的目录路径 | `.apboa/skills` |
| 覆盖策略 | 是否覆盖已有同名技能 | 关闭（跳过） |

**工作原理：**

1. 读取指定目录下的技能包目录
2. 自动规范化：若存在 `skill.md`（小写），自动重命名为 `SKILL.md`
3. 解析各目录中 `SKILL.md` 的 YAML frontmatter 获取 `name`、`description`
4. 将技能包目录安装到平台工作空间 `.apboa/tenants/{tenantCode}/skills/` 下（租户隔离）
5. 将技能包元数据与入库文件持久化到数据库
6. 通过 Redis 发布技能文件同步事件，通知运行节点加载

**目录结构要求：**

```
<指定路径>/
  skill-a/
    SKILL.md          # 须含 YAML frontmatter: name, description
    scripts/           # 脚本文件
    examples/          # 示例文件
    references/        # 引用文件
  skill-b/
    SKILL.md
    ...
```

若指定路径下存在 `skills/` 子目录，系统优先从该子目录加载。当本地路径与平台工作空间路径相同时，系统跳过文件复制，仅更新数据库记录。


## ZIP 压缩包导入详解

**表单字段：**

| 字段 | 说明 | 约束 |
|------|------|------|
| 选择压缩包 | 上传 ZIP 文件 | 仅 `.zip` 格式，≤100MB |
| 标签 | 导入的技能包归入此分类 | ≤6 个字符，支持选择已有或新建 |
| 覆盖策略 | 是否覆盖已有同名技能 | 关闭（跳过） |

压缩包选择区域支持**点击选择**和**拖拽放入**两种交互方式。

**工作原理：**

1. 将上传的 ZIP 文件解压到临时目录（`.apboa/temp/{uuid}/`）
2. 自动解析 `skills/` 根目录（兼容多套一层文件夹的结构，如 `my-skills/skills/...`）
3. 读取解压后目录中的技能包定义
4. 根据覆盖策略安装到工作空间
5. 安装完成后自动清理临时目录

**ZIP 包结构要求：** 同本地装载，支持 `skills/` 在根目录或外层目录下。

:::warning
ZIP 导入过程中的临时目录会在完成后自动清理，不占用额外磁盘空间。
:::


## Git 仓库导入详解

**表单字段：**

| 字段 | 说明 | 约束 |
|------|------|------|
| 技能分类 | 导入的技能包归入此分类 | 默认 `Git导入` |
| 仓库地址 | Git 仓库 URL | 支持 HTTPS 和 SSH 协议 |
| 覆盖策略 | 是否覆盖已有同名技能 | 关闭（跳过） |

**工作原理：**

1. 在 `.apboa/temp/` 下创建 UUID 命名的临时目录
2. 将远程仓库克隆到临时目录
3. 若仓库根目录存在 `skills/` 子目录则优先加载，否则从根目录加载
4. 根据覆盖策略安装到工作空间
5. 安装完成后自动清理临时目录（Windows 下文件占用为正常现象）

**仓库目录结构：** 推荐将技能包统一放置在仓库根目录的 `skills/` 子目录下。

:::info
若仓库中不存在 `skills/` 子目录，系统将仓库根目录视为技能包目录直接加载。
:::


## 文件入库机制

技能包中的文件分为**入库文件**（持久化到数据库）和**纯文件系统文件**（仅存储在磁盘）两类。

### 入库规则

满足以下条件的文件会自动入库到数据库：

1. **根目录的 `SKILL.md`** — 始终入库
2. **`references/`、`examples/`、`scripts/` 目录下的文件** — 扩展名在白名单内时入库

其余文件（如自定义目录下的文件、非白名单扩展名的文件）仅写入文件系统，不入库。

### 文件扩展名白名单

默认白名单包含以下扩展名：

```
md, py, sh, js, ts, json, yaml, yml, xml, txt,
java, cs, go, rs, rb, php, sql, html, css, scss, less, cfg, conf, toml
```

白名单可通过 Params 系统中的 `SKILL_FILE_ALLOWED_EXTENSIONS` 参数进行配置（逗号分隔），修改后自动清除缓存生效。

### 入库文件 vs 纯文件系统文件

| 特性 | 入库文件 | 纯文件系统文件 |
|------|----------|----------------|
| 存储位置 | 数据库 + 文件系统双写 | 仅文件系统 |
| 在线编辑 | 支持 | 支持 |
| 保存方式 | 更新 DB 并同步文件系统 | 仅写入文件系统 |
| 文件树标识 | 显示保存状态图标 | 无状态图标 |
| 删除时 | 同时清理 DB 记录和文件 | 仅删除文件 |


## 文件同步机制

在集群部署环境下，技能包文件需要从 console 节点同步到 runner-file 运行节点。

### 自动同步

以下操作会自动触发文件同步事件（通过 Redis Pub/Sub）：

- **导入技能包**（本地/ZIP/Git） — 发布 `sync` 事件
- **删除技能包** — 发布 `delete` 事件
- **技能包更新** — 通过智能体重注册机制间接同步

### 手动同步

在技能包编辑器底部提供「**同步到运行节点**」按钮，可手动触发当前技能包的文件同步。适用于以下场景：

- 在编辑器中修改文件后希望立即生效
- 集群节点重启后需要重新同步
- 自动同步因网络原因未成功

同步请求发送后，运行节点会自动拉取最新的技能文件并更新本地缓存。


## 在智能体中使用

在智能体配置的「工具与能力」步骤中：

1. 在 **技能包** 区域，按分类浏览可用技能包
2. 勾选需要的技能包（多选）
3. 技能包关联的工具会自动注册到智能体的工具集中

运行时注册流程：
1. 加载智能体关联的所有**已启用**技能包
2. 对每个技能包，读取 SKILL.md 内容和所有入库文件（references/examples/scripts）作为技能知识
3. 查询技能包关联的工具 ID，注册到智能体的 Toolkit 中
4. 在 SKILL.md 内容末尾追加可用工具信息列表，帮助模型感知可用工具

## 最佳实践

### SKILL.md 编写建议

SKILL.md 是模型理解技能能力的关键入口，建议包含：

- **技能目标** — 该技能解决什么问题
- **使用场景** — 什么情况下应该使用该技能
- **操作步骤** — 使用该技能的典型步骤
- **注意事项** — 使用限制和注意点

### 文件管理建议

- 将脚本放入 `scripts/`、示例放入 `examples/`、参考文档放入 `references/` 目录，便于系统自动识别入库
- 二进制资源文件（图片、PDF 等）可上传到任意目录，系统会自动下载引导而非尝试在线预览
- 非白名单文件类型无法在线编辑，请下载到本地修改后重新上传

### 工具关联建议

- 将逻辑上相关的工具组合到同一技能包
- 避免将不相关工具放入同一技能包
- 确保技能包关联的工具都已启用


## 目录结构规范总结

为确保技能包可正常导入和使用，请遵循以下规范：

1. 每个技能包使用独立目录
2. 技能定义文件统一命名为 `SKILL.md`（全大写）；小写 `skill.md` 导入时自动重命名
3. `SKILL.md` 须含 YAML frontmatter，至少包含 `name` 与 `description` 字段
4. 脚本文件放入 `scripts/` 子目录
5. 示例文件放入 `examples/` 子目录
6. 引用文件放入 `references/` 子目录
7. 多个技能包统一放在 `skills/` 父目录下（推荐）


## 权限说明

技能包的创建、编辑、导入、删除等操作需要以下角色之一：

| 角色 | 说明 |
|------|------|
| `TENANT_OWNER` | 租户所有者 |
| `TENANT_ADMIN` | 租户管理员 |
| `TENANT_EDITOR` | 租户编辑者 |

仅查看技能包列表和详情无角色限制。


## 导入后验证

导入完成后建议进行以下检查：

1. 在技能包管理页面确认技能包列表已更新
2. 进入技能包编辑器，确认文件结构完整
3. 检查 SKILL.md 中的 `name` 和 `description` 是否正确解析
4. 在智能体中关联该技能包，验证运行时工具注册是否正常
5. 若为集群部署，确认运行节点已同步技能文件


## 常见问题

### SKILL.md 的 name 和 description 修改后为什么没有同步？

- 编辑 SKILL.md 并保存后，系统会自动解析 YAML frontmatter 中的 `name` 和 `description`，同步到数据库
- 如果没有同步，检查 YAML frontmatter 格式是否正确（`---` 开头和结束）
- 确认已点击保存（Ctrl+S）而非仅编辑未保存

### 导入技能包提示「未识别到技能包」？

- 检查目录结构是否符合规范：每个技能包必须是独立目录，包含 `SKILL.md` 文件
- `SKILL.md` 必须包含 YAML frontmatter，至少包含 `name` 和 `description` 字段
- 如果是 ZIP 导入，确认压缩包内层级正确（支持 `skills/` 在根目录或外层目录下）

### 导入时「跳过」和「覆盖」如何选择？

- **跳过**（默认）：同名技能包保留已有内容，适合防止误覆盖、保留本地修改
- **覆盖**：删除旧文件记录和目录后导入新内容，适合版本更新、全量同步
- 建议：首次导入使用跳过，确认无误后如需更新再使用覆盖

### 如何关联工具到技能包？

- 在技能包卡片右上角菜单中点击「关联工具」，打开双面板选择器
- 左侧面板展示所有已启用的工具，右侧面板展示已关联的工具
- 点击工具即可在左右面板间切换添加/移除
- 两侧均支持关键词搜索

### 多节点部署时技能包文件不同步？

- 在技能包编辑器底部点击「同步到运行节点」按钮，手动触发同步
- 自动同步通过 Redis Pub/Sub 实现，如果 Redis 连接异常可能导致同步失败
- 建议使用 GlusterFS 实现多节点服务器上指定文件夹的实时同步

### 文件树中的文件为什么不能在线编辑？

- 只有扩展名在白名单内的文件才支持在线编辑（如 `.md`、`.py`、`.js`、`.json` 等）
- 非白名单文件（如图片、PDF、二进制文件）会引导下载到本地查看
- 超大文件（>500KB）也会引导下载查看
- 白名单可通过 Params 系统的 `SKILL_FILE_ALLOWED_EXTENSIONS` 参数配置

### SKILL.md 可以删除吗？

- SKILL.md 是技能包的元信息文件，不允许删除
- 在文件树中右键 SKILL.md 不会显示菜单选项
- 如果意外删除了 SKILL.md，需要重新创建并填写 YAML frontmatter

### 技能包关联的工具在智能体中不生效？

- 确认技能包已启用（禁用状态的技能包不会加载）
- 确认关联的工具已启用
- 确认智能体已关联该技能包
- 运行时注册流程：加载已启用技能包 → 读取 SKILL.md 和入库文件 → 注册关联工具到 Toolkit
